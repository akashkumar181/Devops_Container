# Introduction to Jenkins

## What is Jenkins?

Jenkins is an **open-source automation server** used for Continuous Integration (CI) and Continuous Delivery (CD).

It automates software development tasks such as:

- building code
- testing applications
- deployment automation

Jenkins helps developers automate repetitive work in the software development lifecycle. :contentReference[oaicite:2]{index=2}

---

# Why Use Jenkins?

Advantages:

- Automates builds and tests
- Supports CI/CD pipelines
- Integrates with Git, Docker, Maven, Kubernetes
- Large plugin ecosystem
- Cross-platform support

---

# CI/CD with Jenkins

## Continuous Integration (CI)

Automatically:

- integrate code
- build application
- execute tests

after code changes.

---

## Continuous Delivery / Deployment (CD)

Automates application delivery and deployment process.

Workflow:

```text
Code → Build → Test → Deploy
```

---

# Jenkins Setup Basics

Basic setup steps:

1. Install Jenkins
2. Access Dashboard
3. Install Suggested Plugins
4. Create Admin User

Jenkins usually runs on:

```text
http://localhost:8080
```
